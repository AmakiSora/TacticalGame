## 核心胜利策略

### 1. 四据点经济控制（不追求全占）

**关键决策:** 不强行占领敌方腹地的cp_b，而是控制左侧4个据点确保经济优势。

```
我方占领:
  cp_a 西侧补给站 (-4,0) — 第1回合
  cp_n 北部中继站 (0,-4) — 第2回合
  cp_c 中央阵地 (0,0) — 第4回合
  cp_s 南部中继站 (0,4) — 第5回合
  
敌方占领:
  cp_b 东侧补给站 (4,0) — 对方控制
```

**为什么有效:**
- 4个据点 = 60补给/回合 + 10基础 = 70补给/回合
- 对方仅1个据点 = 15补给/回合 + 10基础 = 25补给/回合
- 我方每回合多45补给，10回合多450补给
- 不需要强攻cp_b（深入敌方阵地），减少不必要的伤亡

### 2. 支援单位续航 — 首次部署治疗

**关键决策:** 第9回合部署2个支援单位，为前线游骑兵提供治疗。

```
Turn 9: 部署支援(1,0)和支援(1,4)
Turn 11: 支援(4,-1)治疗游骑兵 — 恢复28HP
```

**为什么有效:**
- 游骑兵75HP/3防，容易被反击打残
- 支援28奶量可以维持游骑兵在前线的作战时间
- 支援的存在让游骑兵海可以持续输出而不需要回撤

### 3. 游骑兵海暴兵 — 从第6回合到第20回合持续部署

**关键决策:** 从第6回合开始每回合都部署游骑兵，持续补充远程火力。

```
游骑兵部署时间线:
  Turn 6: 游骑兵(1,0)
  Turn 10: 游骑兵(5,-1)
  Turn 11: 游骑兵(5,-1)
  Turn 14: 游骑兵(1,0)
  Turn 15: 游骑兵(1,0)
  Turn 16: 游骑兵(1,0)
  Turn 17: 游骑兵(1,0)
  Turn 18: 游骑兵(1,0)
  Turn 19: 游骑兵(1,0)
  Turn 20: 游骑兵(1,0)
  
总计: 10个游骑兵！
```

**为什么有效:**
- 每回合70补给收入，游骑兵75/个，几乎每回合都能部署1个
- 即使前线游骑兵阵亡，后方立即补充，火力不中断
- 最终6个游骑兵同时在场围攻敌方HQ区域

### 4. 侦察兵自杀式突进 — 快速抵达敌方HQ侧翼

**关键决策:** 第10回合派侦察兵(5,-1)→(8,-1)直接突进到敌方HQ侧翼，虽然会被打残但提供了视野和火力。

```
侦察兵(5,-1)→(8,-1): 从地图中央直接跑到敌方HQ门口
```

**为什么有效:**
- 侦察兵移速5，可以快速穿越战场
- 提供敌方HQ附近的视野，让游骑兵可以瞄准攻击
- 即使被打残也完成了侦察任务

### 5. 集火清理 — 优先消灭敌方单位再攻HQ

**关键决策:** 第7-19回合主要攻击敌方单位，不急于攻HQ。

```
Turn 6-8: 集中攻击敌方前线单位
Turn 9-13: 清理敌方中盘部署的单位
Turn 14-19: 清理敌方新部署的单位
Turn 20-21: 围攻敌方HQ
```

**为什么有效:**
- 敌方单位会反击，先清理可以保护己方游骑兵
- 减少敌方单位数量 = 减少己方伤亡
- 清理干净后再攻HQ，可以无压力输出

### 战术效果评估

**结论：** 支援单位有效延长了游骑兵的作战时间，是值得继续使用的战术。但需要注意走位配合。

## 失误与改进

### 失误1: 侦察兵突进过深无保护

**问题:** 第10回合侦察兵(5,-1)→(8,-1)突进到敌方HQ侧翼，周围没有友军支援，被敌方集火打残。
**改进:** 侦察兵突进前应确保有游骑兵在附近提供火力支援，或者由多个单位同时突进分散火力。
**收益:** 避免侦察兵白白牺牲，保留机动单位用于后续任务。

### 失误2: 支援单位走位脱节

**问题:** 支援(1,4)→(4,4)→(7,1)与主力游骑兵走位不一致，治疗距离不够。
**改进:** 支援应部署在游骑兵推进路线上，保持在游骑兵后方2-3格的距离。
**收益:** 确保治疗覆盖所有前线单位，提高治疗效率。

### 失误3: 前期部署过多侦察兵

**问题:** 第1-2回合部署了4个侦察兵（初始1个+部署3个），花费120补给。但侦察兵在中后期战斗力有限。
**改进:** 第1回合部署2个侦察兵即可，第2回合开始部署步兵或游骑兵。
**收益:** 节省补给用于更高效的战斗单位。

### 失误4: 没有更早攻击HQ

**问题:** 整局都在清理敌方单位，直到第20回合才开始围攻HQ。如果更早攻击HQ，可能提前2-3回合结束。
**改进:** 在清理主要威胁后（第15回合左右），可以分出2-3个游骑兵开始攻击HQ，其余继续清理。
**收益:** 提前结束游戏，减少变数。
