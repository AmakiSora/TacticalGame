## 失误与改进

### 失误1: ⚠️ 第2回合整回合空过（最严重）

**问题:** autoplay2.mjs的reachable()函数BFS路径规划失效，第2回合所有move尝试返回invalid_move，导致整回合没有任何有效行动，浪费5个行动点和宝贵的早期发展窗口。
**根因:** BFS的可达性计算逻辑有bug——可能未正确处理"起点格被自身占据"或"已访问集合初始化错误"，导致从单位当前位置出发时所有邻居都被判定不可达。
**改进:** 
- 修复reachable()：确保起点加入dist map时距离为0，邻居判定时排除blocked/occupied但允许起点
- 增加fallback：BFS失败时退化为"尝试6个相邻方向±moveRange内的随机格"
- 移动失败超过3次应立即尝试deploy或其他动作，不要空过整回合
**收益:** 避免空过回合，每回合至少完成3-5个有效动作，预计可提前2-3回合获胜。

### 失误2: 大量move invalid_move（贯穿全局）

**问题:** 日志显示全程有大量"move fail invalid_move"（第1/2/5/6/7/10/11/13/14/15回合都有），说明reachable()返回的"可达格"实际上不可达。
**根因:** BFS虽然排除了blocked terrain，但可能：
- 没排除其他单位占据的格子作为路径中间节点（只能停在空格，但可以经过？需确认规则）
- 半径8边界判定有误：`Math.max(|nq|,|nr|,|nq+nr|) > 8` 对pointy-top是否正确需验证
**改进:** 对照服务器规则确认"移动能否经过友军格子"，修正BFS；对边界用更严格的判定。
**收益:** 移动成功率从约50%提升到90%+，行动点利用率大幅提高。

### 失误3: 第1回合未移动任何单位去占点

**问题:** 第1回合直接部署heavy，但初始的scout/infantry都没动，浪费了抢占cp_b(4,0)的机会。
**改进:** 第1回合应至少用1个行动点让scout→(4,0)占cp_b（紧邻HQ），再部署heavy。
**收益:** 早期多1个据点（+15/回合），经济优势更早建立。

### 失误4: support治疗时机偶尔无效

**问题:** 日志中heal有时显示"->"但没有后续HP（如Turn 4 "heal scout 25 ->"），可能治疗目标已满血或不在射程。
**改进:** 治疗前严格检查 `target.hp < target.maxHp` 且 `hexDist(support,target) <= 1`，避免无效请求。
**收益:** 减少无效治疗，腾出行动点用于攻击或移动。

### 失误5: 重装单位调度效率低

**问题:** 多个heavy在中场反复"move fail"，推进速度慢（heavy移速2，且路径常被堵）。
**改进:** heavy应分批推进，避免3个heavy挤在同一路径互相阻塞；可让1个heavy走北线(-5,-1)，1个走中线，1个走南线分散路径。
**收益:** heavy更快抵达敌方HQ，缩短攻城阶段。

## 游戏机制深度理解

### 1. 重装单位（Heavy）的价值

```
Heavy规格: HP160 / 攻36 / 防16 / 移2 / 射程1 / 花费90 / 不能占点

vs Infantry(HP100/防8): heavy胜，约2-3轮击杀
vs Ranger(射程3): ranger可风筝，但heavy一旦近身ranger必死
vs HQ(防10): heavy每次造成 36-10=26伤害，约9-10次攻击摧毁240HP HQ
```

**关键认知:** heavy移速2是最大短板，长地图（半径8）需要10+回合才能从己方HQ推进到敌方HQ。因此heavy必须**尽早部署**（本局第1回合），否则来不及抵达前线。

### 2. 支援单位（Support）治疗机制

```
Support规格: HP80 / 攻12 / 防5 / 移3 / 射程1 / 花费60 / 治疗量28
治疗射程: 1（必须相邻）
```

**关键认知:** support射程1要求它必须贴身跟随被治疗单位，但support移速3比heavy移速2快，容易脱节。应让support主动靠近前线而非单独行动。

### 3. 经济与据点

```
基础收入: 10/回合
每据点: +15/回合

本局据点收入演变:
  Turn 1-2: 0据点 = 10/回合（仅基础）
  Turn 3+: 占cp_c = 25/回合
  Turn 9+: +cp_a = 40/回合
  中期: +cp_n = 55/回合
  后期: 4据点 = 70/回合

敌方据点收入: 始终0-1据点 = 10-25/回合
收入差: 后期我方70 vs 敌方10-25，约3-7倍碾压
```

### 4. 行动点（Action Points）回顾

```
每回合5点。本局典型分配:
  - 攻击HQ/敌方单位 ×2-3（每激活1单位1点）
  - 移动推进 ×1-2
  - 部署 ×0-1（如有补给）
最佳实践: 让1个单位"移动+攻击"完整做完再激活下一个，最大化每点价值
```

## 总结

### 胜利关键因素
1. **开局爆重装** — 第1回合部署heavy，为长距离推进预留时间（重装移速2需10+回合穿图）
2. **4据点经济** — cp_a/cp_b/cp_c/cp_n 控制，70补给/回合支撑连续重装部署
3. **重装洪流** — 3个heavy（HP160/防16）组成的钢铁阵线，敌方无法击穿
4. **scout突进首攻** — 第12回合scout兵临城下首攻HQ，比第6局早2回合
5. **支援续航** — support治疗保scout存活到完成突进任务

### 核心战术原则

> **"首回合爆重装，四据点养洪流，侦察兵开城门"**

- **重装要早** — heavy移速慢，第1回合就得部署，否则来不及到前线
- **据点驱动经济** — 4据点70/回合，每回合1个heavy/ranger源源不断
- **scout当先锋** — 移速5的scout提前抵达敌方HQ，为heavy主力开攻城先声

### 与之前对局的不同
第7局是GLM5.2-ZC的第二场V2对局，与第6局的关键差异：
1. **重装数量翻倍** — 3 heavy（第6局2 heavy），攻城火力更猛
2. **Player B侧** — 首次从右侧作战，验证策略对称性
3. **scout突进战术** — 首次用scout而非infantry完成首攻HQ
4. **脚本化作战** — 本局全程autoplay2.mjs自动决策（虽然第2回合有bug空过）

### 一句话总结

**第17回合，作为Player B以3重装洪流（首回合即爆）+4据点经济碾压（70 vs 10）+scout第12回合突进首攻HQ，我方HQ无伤摧毁敌方总部，取得V2第七胜、个人两连胜。**
