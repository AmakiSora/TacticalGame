## 核心胜利策略

### 1. 快速抢占控制点（经济优势）

**关键决策:** 第1回合直接派步兵占领cp_b，后续每回合扩张控制区域。

```
Turn 1: 步兵(7,0)→(4,0)占领cp_b — 花费0
Turn 2: 侦察兵(3,2)→(0,0)占领cp_c — 花费0
Turn 4: 侦察兵(0,-1)→(0,-4)占领cp_n — 花费0
Turn 5: 侦察兵(0,-4)→(-4,0)占领cp_a — 花费0
        侦察兵(2,3)→(0,4)占领cp_s — 花费0
```

**为什么有效:**
- 每个控制点提供20金/回合收入，5个全占后每回合收入115金（基础15+100）
- 敌方从第5回合起只有15金/回合基础收入，经济完全碾压
- 高收入让我可以在后期一回合部署9个单位（Turn 13）

### 2. 集中火力逐个击破

**关键决策:** 不分散攻击，每回合集中消灭一个敌方单位。

```
Turn 3: 侦察兵(0,0)+步兵(0,1)+步兵(-1,0) 三人围攻敌方Scout(-1,1) → 击杀
Turn 5: 步兵(-2,1)+步兵(-2,2)+Ranger(-1,1) 三人围攻(-3,2)残血 → 击杀
Turn 7: 步兵(-3,1)+Ranger(-1,1) 围攻(-4,1) → 击杀
Turn 9: 步兵(-6,1)+Ranger(-4,1)+Heavy(-5,0)+Scout(-5,-1) 围攻(-4,-1)Scout → 击杀
```

**为什么有效:**
- 每个单位攻击伤害约20-35，单个单位难以击杀满血敌人（100HP）
- 集中2-3个单位可以确保一回合内击杀
- 减少敌人反击机会，保护己方单位

### 3. 兵种协同（前排+远程+治疗）

**关键决策:** 步兵扛线，Ranger远程输出，Support治疗，Heavy攻坚。

```
阵型示例(Turn 8):
  前排: Infantry(-5,1), Infantry(-6,2), Heavy(-5,0)
  中排: Ranger(-3,1), Support(-1,1)
  后排: Ranger(7,0), Scout(7,1), Heavy(2,1)
```

**为什么有效:**
- Infantry（28攻/8防/100HP）性价比高，适合前排
- Ranger（42攻/3防/75HP/射程3）是核心输出，后期大量部署
- Heavy（36攻/16防/160HP）适合攻坚和抗线
- Support（28奶量）保持部队续航

### 4. 利用敌方AI不操作的优势

**关键决策:** 敌方AI每回合直接结束，利用免费回合快速部署和推进。

```
Turn 5: 敌方获得35金却不做任何操作，直接结束
Turn 6: 同上
Turn 7+: 持续如此
```

**为什么有效:**
- 敌方单位不移动、不攻击，我可以无压力推进
- 但也要注意：敌方会在回合开始自动获得收入并可能部署新单位
- 每回合结束时要注意清理敌方新部署的单位

### 5. 后期经济碾压式暴兵

**关键决策:** 占领全部5个控制点后，利用115金/回合收入大量暴兵。

```
Turn 13(一回合内):
  部署: Ranger×8 + Infantry×1 = 560金
  剩余: 120金
  总单位数: 23
```

**为什么有效:**
- 5控制点 = 100金/回合收入，是基础15金的6.7倍
- 后期不需要节约，可以全力暴兵快速结束游戏

## 失误与改进

### 失误1: 距离计算错误导致移动浪费

**问题:** Turn 4试图让步兵(4,0)→(1,0)，但路径被(2,0)障碍物阻挡，浪费了一次行动。
**改进:** 移动前先检查路径上的障碍物（blocker terrain），绕路走安全路线。
**收益:** 避免无效操作，每回合节约一次移动机会。

### 失误2: 单位ID追踪混乱

**问题:** 敌方AI每回合会部署新单位，导致旧ID失效。多次出现"target not found"错误。
**改进:** 每次攻击前重新获取目标ID，或使用AI脚本的自动瞄准功能。
**收益:** 避免攻击失败浪费行动次数。

### 失误3: 侦察兵被反击至残血

**问题:** Turn 7，侦察兵(-6,0)攻击敌方步兵后被反击打至24/70HP。
**改进:** 侦察兵防御低（4防），不应主动攻击高攻目标（步兵28攻）。应优先用步兵或重装攻击。
**收益:** 保留侦察兵的机动性和占领能力。
