## 失误与改进

### 失误1: 早期浪费行动点试图治疗满血单位

**问题:** 第4回合尝试用支援兵治疗游骑兵（75/75满血），返回"target out of range"后才意识到目标满血。
**改进:** 治疗前应检查 `target.hp < target.maxHp - 5`，避免对满血单位发治疗请求。
**收益:** 节省行动点和网络请求，每回合多发1次有效攻击。

### 失误2: 路径规划忽略 water/blocker 地形

**问题:** 第3回合游骑兵(-3,0)→(-1,0)失败（被中央blocker(0,0)阻挡），支援兵(0,-3)→(0,-2)失败（water），多次重试浪费决策时间。
**改进:** 移动前应预计算路径，绕开 water(-1..1,-2)(-1..1,2) 和 blocker(-2,0)(2,0)(-5,-2)(5,2)。pointy-top六边形相邻6方向为 (1,0)(1,-1)(0,-1)(-1,0)(-1,1)(0,1)。
**收益:** 避免无效移动尝试，每行动点都用到位。

### 失误3: 部署位置选择错误（非相邻）

**问题:** 第3回合尝试从cp_n(0,-4)部署到(1,-3)，返回"deploy target must be adjacent"。(1,-3)不是cp_n的六边形邻居。
**改进:** 部署前确认目标格是据点的6个相邻格之一，可用方位数组 [(1,0),(1,-1),(0,-1),(-1,0),(-1,1),(0,1)] 偏移计算。
**收益:** 部署成功率100%，不浪费行动点。

### 失误4: 误判单位每回合攻击次数

**问题:** 第4-5回合多次尝试让同一单位二次攻击（"already acted this turn"），误以为游骑兵激活后可多次攻击。
**改进:** 每单位每回合只能攻击1次（即使已激活可免费移动，但攻击仍限1次）。激活只让"移动"免费，不增加攻击次数。
**收益:** 准确预判行动预算，规划更高效的攻击序列。

### 失误5: 攻击HQ略晚，可更早结束

**问题:** 第13回合步兵已到(6,0)（HQ dist 2），但第14回合才开始攻HQ。其实第13回合部分单位已在射程内却选择移动而非攻击。
**改进:** 当单位进入敌方HQ射程（步兵dist≤1，游骑兵dist≤3）时，应优先攻击而非继续移动占位。
**收益:** 可能提前1-2回合结束，减少敌方反击窗口。

## 游戏机制深度理解

### 1. 行动点（Action Points）机制

```
每回合行动点: 5点
- 激活新单位（首次移动/攻击/治疗/部署）: 消耗1点
- 已激活单位后续动作（如移动后攻击）: 免费
- 部署: 始终消耗1点（新单位本回合不能再移动）
- 达到5点上限后: 已激活单位可继续动作，但不能激活新单位
```

**关键认知:** "激活"是按单位计而非按动作计。规划时应优先让一个单位完整做完"移动+攻击"再用下一个，最大化每点行动点的价值。

### 2. 六边形距离公式（pointy-top axial）

```
dist = (|dq| + |dq+dr| + |dr|) / 2
     = max(|dq|, |dr|, |dq+dr|)

例: (0,-1)→(1,1): dq=1,dr=2, dist=(1+3+2)/2=3 ✓（游骑兵射程内）
   (0,0)→(1,1):  dq=1,dr=1, dist=(1+2+1)/2=2 ✗（超出步兵射程1）
```

**实战教训:** 不要靠直觉判断相邻关系，必须用公式计算。我因误判(0,0)与(1,1)的距离导致攻击失败。

### 3. 经济模型

```
基础收入: 10补给/回合
每个据点: +15补给/回合（回合开始时结算）

3据点收入: 10 + 45 = 55/回合
4据点收入: 10 + 60 = 70/回合
5据点收入: 10 + 75 = 85/回合

结论: 3据点已足够碾压0据点对手（5.5倍收入），无需强求5据点全占。
      据点越多边际收益递减，但机会成本（兵力用于争夺）上升。
```

### 4. 单位性价比

```
游骑兵(75): 攻42/射程3 — 远程压制之王，性价比最高
重装(90):   HP160/防16 — 难以击杀的前线核心
步兵(45):   攻28/占点 — 占点工具人，中后期战斗力下降
侦察兵(40): 移5/占点 — 快速占点和深入敌后
支援(60):   治疗28 — 续航核心，但需贴身（射程1）
```
