## 核心胜利策略（三局共性）

### 1. 开局书（Opening Book）抢三据点 — 最关键的改进

**决策:** autoplay3 的开局书让 Player A 首回合固定执行 `scout→(-4,0)占cp_a + 双步兵推进`，确保前3回合拿下 cp_a/cp_n/cp_c。

```
三局开局对比（我方首次三据点成型回合）:
  第8局: t1 cp_a, t3 cp_c, t5 cp_n+cp_b → t5 四据点
  第9局: t2 cp_b, t4 cp_c, t6 cp_s → t6 三据点（后手慢1回合）
  第10局: t1 cp_a, t3 cp_c, t5 cp_n+cp_b → t5 四据点
```

**为什么有效:**
- 3据点 = 45g/回合 + 10基础 = 55g/回合，对手若0-1据点仅10-25g/回合，**收入差2-5倍**
- 经济优势在第6-8回合转化为兵力碾压，是不可逆的滚雪球
- 相比我早期手动对局（第2回合因 BFS bug 空过），开局书彻底消灭了"起步落后"

### 2. 修正 BFS 路径规划 — 从"半残"到"全功能"

**决策:** 放弃我旧脚本 bugg 的 reachable()，改用仓库 `skill/ai-player.mjs` 的正确算法。

```
旧脚本(autoplay2): move 成功率约50%，第7局第2回合整回合空过
新脚本(autoplay3): reachableCells() 用 isValidHex(radius)+isEmptyPlain 正确判定
  → move 成功率接近100%，每个行动点都用在刀刃上
```

**为什么有效:**
- 正确的六边形距离公式 `max(|dq|,|dr|,|ds|), s=-q-r`，不再误判射程
- BFS 排除 water/blocker/unit/HQ，移动只发合法请求，消灭 invalid_move
- 第10局能15回合速胜，根本原因是行动点零浪费

### 3. 评估函数驱动的贪心决策

**决策:** 引入 `evaluate(state)` 评分（补给+据点+我方HP-敌方HP-敌方HQ血+攻击机会），每个候选动作模拟后选 Δscore 最大。

```
score = 补给×1 + 我据点×80 - 敌据点×70 + Σ我HP×2 - Σ敌HP×3
        - 敌HQ剩余HP + 我HQ剩余HP + 我方可攻击单位数×15 + 我方单位数×10
```

**为什么有效:**
- 击杀预判：`willKill` 加 +5000 优先级，残血单位必补刀（三局 unit_death 频繁，多为我方击杀）
- 移动预判：优先选移动后能攻击敌方/HQ 的格子，避免"移动完发现射程不够"
- 不盲目占点：评估函数权衡占点收益与暴露风险

### 4. Support 海续航 — 针对 dsv4 远程对射的关键

**决策:** 三局我方 support 部署数（4/4/1）显著，靠密集治疗在 ranger 对射中耗赢对手。

```
第8局: DEP support×4，治疗保住前线 ranger
第9局: heal×9（三局最多），ranger 持续作战不掉线
第10局: support×1 + ranger×3，少奶多输出速攻
```

**为什么有效:**
- dsv4-Script 也是 ranger 海战术，双方远程对射伤害接近，**谁奶多谁赢消耗战**
- support 治疗量28-35/次，相当于抵消敌方1次攻击，续航差距累积成胜利
- 第9局 heal×9 是制胜关键，敌方 heal 仅个位数

### 5. Ranger 海终局收割

**决策:** 中后期把经济转化为 ranger 堆叠（3/2/3），三路包抄敌方 HQ。

```
第8局: ranger×3 从(5,0)推进，18回合围杀B HQ
第9局: ranger×2 从(-1,0)(-5,0)深入A侧，17回合杀A HQ
第10局: ranger×3 从(0,3)(1,4)(5,0)三面包围，15回合速杀
```

**为什么有效:**
- ranger 射程3可在敌方步兵反击范围外输出，零损耗消耗
- 多 ranger 集火，敌方任何单位2-3轮必死
- 攻 HQ 时多 ranger 齐射，每回合 ~100+ 伤害，2-3回合摧毁 240HP HQ

## 失误与改进

### 失误1: 据点反复易主，争夺效率低

**问题:** 三局 cp_c/cp_b/cp_a 都出现5-10次反复占领（如第8局 cp_c 被夺回6次）。占点单位站位后常被敌方击杀导致丢点。
**改进:** 关键据点（cp_c 中央）应常驻1个 heavy 或步兵防守，而非占领后立刻离开。评估函数应给"守点"额外加分。
**收益:** 减少据点拉锯回合，预计每局可再快2-3回合。

### 失误2: support 走位偶尔脱节

**问题:** 第8/9局 support 部署在(1,0)(5,0)等后方，有时跟不上前线 ranger，治疗射程1的要求导致部分回合治疗无效。
**改进:** support 应主动向最受伤的友军移动（pickMove 中 support 的 goal 已设为最伤友军，但执行不够激进）；可让 support 移速3优势充分利用。
**收益:** 治疗命中率提升，ranger 存活率进一步提高。

### 失误3: 第10局 support 部署偏少（仅1个）

**问题:** 第10局只部署1个 support，虽然速胜但中段若敌方反扑，ranger 续航会不足。属于"赌速攻"的激进配置。
**改进:** 即使速攻，至少保2个 support；评估函数应在"多伤员且 support<2"时强制优先部署 support（已实现 unshift，但触发条件可放宽）。
**收益:** 稳健性提升，应对敌方反扑更有底气。

### 失误4: 开局书仅适配 Player A 默认布局

**问题:** `matchesDefaultStartA` 只在 Player A + 初始3单位在(-7,0)(-7,1)(-7,-1)时触发。第9局我是 Player B，开局书失效，靠通用启发式慢了1回合。
**改进:** 为 Player B 写镜像开局书（scout→(4,0)占cp_b + 双步兵西推）。
**收益:** 后手局也能前3回合抢3据点，消除先后手差距。

### 失误5: 攻击目标选择未考虑反击威胁

**问题:** 评估函数的攻击优先级（support>ranger>scout>infantry>heavy）只看目标价值，未考虑"打这个目标会不会被它旁边的敌方单位反击"。
**改进:** 攻击前评估"攻击后我方单位是否暴露在更多敌方射程内"，引入威胁惩罚项。
**收益:** 减少贪攻击导致的单位阵亡。

## 游戏机制深度理解（三局验证）

### 1. 经济模型 — 据点是胜负手

```
基础收入: 10g/回合
每据点: +15g/回合

3据点: 55g/回合（碾压0据点对手5.5倍）
4据点: 70g/回合（碾压2倍于2据点对手）
5据点: 85g/回合（边际收益递减，5-action限制下花不完）

三局验证: 我方首次达到3据点的回合(5/6/5)后，胜势基本锁定
```

### 2. 单位对位（ranger vs ranger 海时代）

```
ranger(75g): HP75/攻42/防3/移2/射程3 — 远程对射之王
support(60g): HP80/攻12/防5/移3/射程1/治疗28-35 — 续航核心
scout(40g): HP70/攻18/防4/移5/射程1 — 占点+侦察
heavy(90g): HP160/攻36/防16/移2/射程1 — 终结者（本三局未重点用）
infantry(45g): HP100/攻28/防8/移3/射程1 — 占点肉盾

ranger海时代的制胜公式: ranger(输出) + support(续航) + scout(占点)
```

### 3. 行动点机制回顾

```
每回合5点。三局典型分配:
  - 攻击 ×2-3（击杀/消耗）
  - 治疗 ×0-2（残血时）
  - 移动/占点 ×1-2
  - 部署 ×0-1
最佳实践: 一个单位"移动+攻击"做完再激活下一个，单点价值最大化
```

## 总结

### 胜利关键因素
1. **开局书抢三据点** — 前3回合锁定 cp_a/cp_n/cp_c，经济碾压从第1回合奠定
2. **修正 BFS 路径规划** — 行动点零浪费，move 成功率接近100%（对比旧脚本~50%）
3. **评估函数贪心决策** — 击杀预判+移动预判，每个动作选 Δscore 最大
4. **support 海续航** — 针对 dsv4 ranger 海，密集治疗赢消耗战（第9局 heal×9 制胜）
5. **ranger 海终局收割** — 经济转化为 ranger 堆叠，三路包抄摧毁敌方 HQ

### 核心战术原则

> **"开局书锁三据点，修正BFS零浪费，support续航耗赢ranger海"**

- **开局标准化** — 开局书消除人为失误，首回合必抢 cp_a
- **基础设施要正确** — 路径规划/距离公式是底层，错了再好的战术也打折
- **对位决定配置** — 敌方 ranger 海，我方必须 support+ranger 对等且奶量占优
- **攻防不对称** — 射程3+正确走位让我方HQ零受损，敌方HQ必被摧毁

### 与之前对局的不同
本三局（autoplay3 脚本）相比之前的最大进化：
1. **从手动/半自动到全自动脚本** — autoplay3 全程自动决策，无需人工干预
2. **从启发式到评估函数** — 引入 Δscore 选择，决策更理性
3. **从怕 ranger 海到压制 ranger 海** — support 海战术专门克制 dsv4 的 ranger 海
4. **击败了此前不可战胜的对手** — dsv4-Script 五连胜被终结

### 一句话总结

**三局连胜（18/17/15回合）横扫 V2 霸主 dsv4-Script，靠"开局书抢据点+修正BFS零浪费+support海续航ranger海收割"体系，我方HQ全程240未损，敌方HQ三局全灭。**
